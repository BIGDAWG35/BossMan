## Common Informal Betting Formats in Baseball

### Baseball Squares / Grid Pools
Less common than football, but can be adapted:
- 5x5 or 10x10 grid
- Axes = last digits of runs for each team
- Payouts at end of each inning or final score

### Typical Setup:
- Single game or short series (3-game set)
- Last digits of team scores determine winners
- Prizes split by halves (end of 5th inning, final)

### Prediction Pools
- Predict exact final score or run differential
- Points for accuracy
- Multiple rounds (series-wide)

### Over/Under Bets
- Wager on total runs (over/under)

### Inning-by-Inning Props
- Which team scores in given inning
- Total runs per inning
- Number of pitchers used

### Season/Series Long Pools
- Track predictions across series or week
- Points for correct outcomes

### Fantasy-Style Draft Pools
- Draft players
- Earn points based on stats

## How to Set Up Baseball Squares

### Choose Grid Size:
| Size | Squares | Use |
|------|---------|-----|
| 5x5 | 25 | Small groups |
| 10x10 | 100 | Larger groups |
| 3x3 | 9 | Very small |
| 6x6 | 36 | Medium |

### Scoring Labels:
- Use last digit of runs for each team
- Home team vs Visitor

### Steps:
1. Choose grid size
2. Set price per square
3. Define prize pool
4. Sell squares - track ownership
5. Randomize digits AFTER all sold
6. Set payout rules (inning vs final)
7. Share grid with players
8. Determine winners
9. Distribute prizes

### Payout Example (5x5, $2/square):
- Total: $50
- End of game: 60% = $30
- Inning payouts: 40% = $20

## Best Practices

### Transparency
- Publish grid, ownership, rules BEFORE selling

### Communication
- Use single channel (group chat, shared doc)

### Documentation
- Track: Square #, Player, Price, Digits, Winners

### Notes:
- Baseball scores can be low early
- Consider inning-based payouts
- Extra innings - decide in advance how handled
