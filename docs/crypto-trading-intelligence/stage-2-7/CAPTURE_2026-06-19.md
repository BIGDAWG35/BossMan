## What failed

### Failure 1 — Perplexity Browser QA path dead (whole-pipeline block)

**Symptom:** `browser_navigate https://www.perplexity.ai` returns AX tree with zero bounds. No element to click, no usable interaction.

**Root cause:** Hermes CUA daemon / Perplexity desktop app both have a known zero-bounds capture bug. Documented since 2026-05-23.

**Resolution:** Permanently route around it. Spec says "Browser QA primary, fallback Brave". Updated the actual ranking after empirical testing:
1. Perplexity Browser QA — DEAD (CUA bug)
2. Brave Search via curl — 429 rate-limited
3. Perplexity Search API — forbidden by locked rule (no API key by design)
4. DDG / Searx / Kagi free tier — all gated or empty
5. **Internal derivation from Stage 1 + Stage 3 — WORKED**

**Status:** Open (degraded). PARTIAL is the honest ceiling until CUA recovers.

### Failure 2 — `daily_research.js` schema mismatch with `daily_memo.js`

**Symptom:** `daily_decision.js:117` reads `memo.research_quality` and reports `MISSING` even when Stage 2 ran successfully. `daily_memo.js:100-189` reads research differently than `daily_research.js` writes it.

**Root cause:** Schema drift between Stage 2 producer and Stage 3/4 consumer. Old Stage 2 wrote `research[coin_index].research` (array-keyed); new memo expects `research.research[symbol]` (text map) + `research.symbols[]` + `research.coins_covered`.

**Resolution:** Extended `finalize()` to write both legacy + memo-compatible shapes. One-time migration; idempotent on re-run.

### Failure 3 — DeepSeek mislabeling — `daily_memo.js:169-171`

**Symptom:** DeepSeek was told research came from "Perplexity Search Pro" even when it was internal-only (no external fetch happened). Result: DeepSeek rated `research_quality: MISSING` for valid internal research.

**Root cause:** `buildPrompt()` hardcoded a fixed source label regardless of `research.source`.

**Resolution:** Rewrote `buildPrompt()` with 3-way switch on `research.source`. Extended instruction set so PARTIAL is recognized as a valid rating.

### Failure 4 — `do_not_touch` schema violation (CAUGHT IN QA)

**Symptom:** DeepSeek emitted `"MEMECOINS WITH LOW VOLUME AND HIGH VOLATILITY (E.G., SPXUSDT, 1000MOGUSDT)USDT"` as a single string in `do_not_touch`. Without sanitization, the bot's decision file would contain malformed data and downstream `do_not_touch || []` calls would silently break.

**Root cause:** DeepSeek can emit sentence fragments instead of JSON arrays for symbol lists, especially under instruction pressure. Pre-existing `cleanSym` only uppercased and appended `USDT` — it accepted ANY string.

**Resolution:** Two-layer fix (defense in depth):
- **Producer (`daily_memo.js`):** `sanitizeMemo(parsed, radar)` validates every symbol against `/^[A-Z0-9]{2,15}USDT$/`. Anything else is dropped with a warning logged to stderr.
- **Consumer (`daily_decision.js`):** Replaced `cleanSym` with strict regex filter. Only valid USDT pairs survive.

### Failure 5 — `daily_pipeline.sh` BSD `date` doesn't support `%3N` (caught during first end-to-end run)

**Symptom:** `date +%s%3N` returned `date: illegal time format` and bash treated `17818990883N` as a numeric literal, choking the elapsed-time math.

**Root cause:** macOS ships BSD `date` which does not support `%3N` (millisecond precision). `gdate` from coreutils is not installed.

**Resolution:** Replaced with Python `time.time()*1000`. <10ms overhead, cross-platform-safe.

## Root-cause themes

1. **Spec vs reality gap.** The original spec assumed "Perplexity Browser QA primary". The host cannot run it. Resolution: lock in internal-derivation as the honest path and document PARTIAL as the ceiling.
2. **Schema drift between stages.** Stages 2→3→4 wrote/read slightly different shapes. Resolution: extended `finalize()` to emit both legacy and memo-compatible shapes.
3. **Honest labeling of LLM inputs.** Producer scripts must label sources truthfully, or downstream LLMs make wrong judgments. Resolution: 3-way `source` taxonomy + per-source prompt sections.
4. **LLM output as data structure.** DeepSeek occasionally emits sentences instead of JSON arrays for symbol lists. Resolution: producer-side sanitizer + consumer-side regex filter (defense in depth).
